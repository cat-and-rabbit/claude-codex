Claude Code のプロンプトに `/` を入力すると、コマンドの一覧が出てくる。これらは Claude への指示ではなく、**Claude Code 自体に対する操作**である。たとえば「会話を整理する」「巻き戻す」「終了する」といった、対話の外側の動きを担う。

組み込みのコマンドは多数あるが、本章では日々の作業で繰り返し使う基本セットに絞って紹介する。

## まず覚えるもの

| コマンド | できること |
|---|---|
| `/help` | 使えるコマンドの一覧と説明を見る |
| `/clear` | 新しい会話を始める。文脈を全部リセット |
| `/compact` | 会話を要約して、コンテキストの空きを増やす |
| `/init` | プロジェクトに CLAUDE.md の雛形を生成する |
| `/rewind` | 過去の状態 (会話・コード) に巻き戻す |
| `/diff` | 未コミットの変更を対話的に確認する |
| `/permissions` | 許可ルールの確認・編集 |
| `/resume` | 過去の会話を再開する |
| `/model` | 使うモデルを切り替える |
| `/usage` | セッションのコストや利用量を確認する |
| `/exit` | Claude Code を終了する |

迷ったら `/` だけ打って一覧を眺めれば、必要そうなものが見つかる。

## よく使うものを中心に

### 会話を仕切り直す ─ `/clear`

別の話に切り替えたい時、これまでの文脈をリセットして新しい会話を始める。長いセッションに無関係なやり取りが残ったままだと Claude の判断が引きずられがちなので、タスクが切り替わるタイミングで使う癖を持つとよい。

過去の会話自体は消えない。`/resume` で呼び戻せる。

### コンテキストを圧縮する ─ `/compact`

同じ会話を続けたいけれど、ここまでのやり取りでコンテキストが満杯になってきた、という時に使う。Claude がそれまでの内容を要約し、空きを作る。引数を渡すと要約の重点を指定できる。

```
/compact 認証フロー周りを優先して残して
```

`/clear` との違いは、**続きを話せるかどうか**。タスクを変えるなら `/clear`、続けるなら `/compact`、と覚えておく。

### 巻き戻す ─ `/rewind`

うっかりした変更を取り消したい、間違った方向に進みすぎた、という時に過去のチェックポイントに戻れる。会話だけ戻す・コードだけ戻す・両方戻す、を選べる。`Esc` を 2 回押しても同じメニューが開く。

ただし、Claude が触っていない外部の変更 (手で書いた差分、CI、ビルド成果物など) は対象外なので、git の代わりにはならない。

### 雛形を作る ─ `/init`

新しいプロジェクトで最初に走らせると、Claude がコードベースを読んで CLAUDE.md の下書きを作ってくれる。前章「CLAUDE.md で前提を渡す」で触れた通り、ここから自分の手で削り・足して整える。

### 確認する系 ─ `/diff` `/permissions` `/usage`

- `/diff` ─ 未コミットの変更を対話的に閲覧。Claude の積み重ねた変更を一気に眺めるのに便利
- `/permissions` ─ 許可済みのコマンドや拒否ルールを確認・編集する
- `/usage` ─ そのセッションでどれくらい使ったか、料金プランの利用量を確認する (`/cost` `/stats` は別名)

## その先

ここで紹介したのは組み込みコマンドのうち、特に頻度の高いものに限ってある。実際にはもっと多く、サブエージェントの管理 (`/agents`)、MCP サーバー (`/mcp`)、フック (`/hooks`)、計画モード (`/plan`)、サンドボックス (`/sandbox`) など、機能ごとに専用のコマンドが用意されている。

さらに、**バンドルスキル**と呼ばれるコマンド群もある。`/simplify` `/debug` `/loop` などがそれで、固定ロジックのコマンドではなく、Claude に手順書を渡して動かす仕組みになっている。自分で作ることもできる。

これらは応用の章で改めて扱う。基礎としては「上の表のコマンドが手に馴染んでいる」状態を目指せば十分である。

---

なお、上の表に挙げた `/clear` と `/compact` は、**長い会話とどう付き合うか**という問題に直結する。次章ではその観点から、両者の使い分けをもう一段掘り下げる。
